Einführung

Eine API (Application Programming Interface) ist eine Schnittstelle, die es zwei Softwareprogrammen erlaubt, miteinander zu kommunizieren. Sie stellt Funktionen, Daten und Dienste zur Verfügung, die von anderen Programmen genutzt werden können, ohne dass man den Quellcode der Anwendung kennen muss.

Beispiel: Eine Wetter-App ruft aktuelle Wetterdaten über eine API eines Wetterdienstes ab – sie fragt also beim „Wetter-Server“ nach.

APIs werden in fast allen modernen Anwendungen genutzt – ob beim Online-Shopping, beim Login mit Google oder beim Versenden von Nachrichten.

Arten von API's

Es gibt verschiedene Typen von APIs, je nach Einsatzzweck:

1. REST (Representational State Transfer)

  • Weit verbreitet im Web.
  • Nutzt HTTP-Methoden wie GET, POST, PUT, DELETE.
  • Arbeitet meist mit JSON als Datenformat.

2. SOAP (Simple Object Access Protocol)

  • Älteres Protokoll.
  • Nutzt XML zur Kommunikation.
  • Komplexer, aber stark typisiert und sicherer in bestimmten Umgebungen.

3. GraphQL

  • Flexibles API-System von Facebook.
  • Ermöglicht, genau die Daten abzufragen, die man benötigt.
  • Moderne Alternative zu REST, aber etwas komplexer in der Einrichtung.
REST API Aufbau

Eine typische REST-API besteht aus sogenannten Endpunkten, also bestimmten URLs, über die Daten abgerufen oder gesendet werden.

HTTP-Methoden:

  • GET – Daten abrufen (z. B. Nutzer anzeigen)
  • POST – Neue Daten anlegen (z. B. Nutzer registrieren)
  • PUT – Vorhandene Daten vollständig aktualisieren
  • PATCH – Teilweise Aktualisierung
  • DELETE – Daten löschen

Beispiel:

http

GET https://api.example.com/users

POST https://api.example.com/users

HTTP-Statuscodes:

  • 200 OK – Alles erfolgreich
  • 201 Created – Neue Ressource erfolgreich erstellt
  • 400 Bad Request – Ungültige Anfrage
  • 401 Unauthorized – Zugriff verweigert (z. B. fehlender API-Key)
  • 404 Not Found – Ressource nicht gefunden
  • 500 Internal Server Error – Fehler auf dem Server
JSON Format

JSON steht für JavaScript Object Notation und ist das Standardformat für Datenaustausch bei Web-APIs. Es ist leicht lesbar und einfach in vielen Programmiersprachen zu verwenden.

Beispiel:

json

{
"name": "Max",
"alter": 25,
"interessen": ["Programmieren", "Lesen", "Sport"]
}

Jede API-Antwort besteht meistens aus einem JSON-Objekt mit strukturierten Daten.

Beispiel Request

Hier ist ein einfacher API-Request mit JavaScript und der fetch()-Funktion:

javascript

fetch("https://api.example.com/users")
.then(response => {
if (!response.ok) {
throw new Error("Fehler beim Laden der Daten");
}
return response.json();
})
.then(data => {
console.log(data);
})
.catch(error => {
console.error("Fehler:", error);
});


Dieser Code ruft Daten von einer API ab und gibt sie in der Konsole aus. Das ist der Standardweg, um APIs im Browser anzusprechen.

Sicherheit

APIs dürfen nicht für jeden öffentlich zugänglich sein. Es gibt mehrere Sicherheitsmechanismen:

1. API-Keys

Ein API-Key ist ein geheimer Schlüssel, den du bei Anfragen mitschickst:

http

GET https://api.example.com/data?apikey=123456

2. OAuth

Ein modernes Authentifizierungssystem, z. B. für „Login mit Google“. Der Nutzer gibt einer App Zugriff auf bestimmte Daten.

3. CORS (Cross-Origin Resource Sharing)

Browser-Sicherheitsfunktion, die regelt, welche Webseiten auf eine API zugreifen dürfen.

4. Rate Limiting

Begrenzt, wie oft ein Nutzer in einer bestimmten Zeit Anfragen senden darf – zum Schutz vor Missbrauch.

Fazit

Eine API ist wie ein Kellner in einem Restaurant: Sie nimmt deine Bestellung entgegen (z. B. „Zeig mir die Nutzerdaten“), leitet sie an die Küche (den Server) weiter und bringt dir das fertige Gericht (Daten) zurück.

Wenn du verstehst, wie APIs funktionieren, kannst du Websites mit externen Daten verbinden, eigene Services anbieten oder komplexe Anwendungen bauen. APIs gehören zum Handwerkszeug jedes modernen Entwicklers!
